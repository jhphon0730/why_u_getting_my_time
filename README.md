# 테스트 케이스 & 검증 결과 관리 웹

## 목적

프로젝트에서 매주 회의를 통해 관리하던
**테스트 Action Item을 웹에서 한눈에 관리**하기 위한 시스템이다.

테스트 진행 상황, 결과, 증빙을 하나의 화면에서 확인할 수 있도록 하여
회의와 관리에 드는 비용을 줄이는 것을 목표로 한다.

---

## 배경

### 기존 방식

* 테스트를 Action Item으로 관리
* 진도점검 회의에서 항목을 하나씩 확인
* 엑셀, 문서, 메일 등으로 정보가 분산됨
* 테스트 증빙 자료를 직접 찾아 확인해야 함

### 문제점

* 회의 시간이 불필요하게 길어짐
* 테스트 상태 파악에 많은 커뮤니케이션이 필요함
* 프로젝트 관리자가 전체 진행 상황을 한눈에 파악하기 어려움

---

## 개선 방향

* 테스트 Action Item을 웹에서 통합 관리
* 회의 시 대시보드 화면 하나로 전체 상황 공유
* 테스트 결과와 증빙 자료를 한 곳에서 관리

이를 통해
**회의는 줄이고, 상태 파악은 빠르게** 하는 것을 목표로 한다.

---

## 핵심 개념

### 프로젝트

* 하나의 업무 단위
* 여러 개의 테스트 케이스(Action Item)를 포함

### 테스트 케이스

* 회의에서 관리하던 Action Item에 해당
* 누가, 언제까지, 어떤 테스트를 수행하는지 관리

### 테스트 실행 결과

* 테스트를 실제로 수행한 기록
* 실행 결과와 증빙 자료를 함께 저장
