# API

## 공통 사항

- 사용자 인증 방식 : JWT
- 송수신 프로토콜 : JSON
- 모든 API는 프로젝트 단위로 동작

---

## 인증 / 사용자 API

- 사용자 등록
- 사용자 조회
- 로그인 / 로그아웃
- - 사용자의 권한은 사용자 자체가 아닌
    프로젝트 참여(ProjectMember)를 통해 결정함

ETC.

- User는 사용자 정보만 관리
- 권한 판단은 항상 프로젝트 기준

---

## 프로젝트 API

- 프로젝트를 생성하고 관리
- 프로젝트 참여자를 관리
- 프로젝트 생성
- 프로젝트 목록 조회
- 프로젝트 상세 조회
- 프로젝트 수정
- 프로젝트 참여자 추가
- 프로젝트 참여자 제거
- - 사용자는 프로젝트 생성자 또는 참여자로서
    프로젝트마다 다른 권한을 가질 수 있음

연관 테이블:

- Project
- ProjectMember

---

## 테스트 케이스 상태 API

- 프로젝트별 테스트 케이스 상태를 관리
- 상태의 활성 여부를 관리
- 프로젝트의 테스트 케이스 상태 목록 조회
- 테스트 케이스 상태 추가
- 테스트 케이스 상태 수정 (이름)
- 테스트 케이스 상태 비활성화
- - 테스트 케이스 상태는 프로젝트별로 독립적으로 관리됨

연관 테이블:

- TestStatus

ETC.

- 프로젝트마다 다른 업무 흐름을 허용
- 상태 커스터마이징 지원

---

## 테스트 케이스 API

- 테스트 Action Item을 관리
- 테스트의 현재 상태와 담당자를 관리
- 테스트 케이스 생성
- 테스트 케이스 목록 조회
- 테스트 케이스 상세 조회
- 테스트 케이스 수정
- 테스트 케이스 상태 변경
- 테스트 케이스 담당자 변경

ETC.

- 상태 변경은 일반 수정 API와 분리
- 담당자 변경 역시 이력 관리를 위해 분리

연관 테이블:

- TestCase
- TestStatus
- User

---

## 테스트 케이스 이력 API

- 테스트 케이스의 변경 이력을 관리

- 감사 및 추적 목적의 조회 전용 API
- 테스트 케이스 상태 변경 이력 조회
- 테스트 케이스 담당자 변경 이력 조회

연관 테이블:

- TestStatusHistory
- TestAssigneeHistory

ETC.

- 현재 상태와 과거 기록을 분리
- 관리 및 감사 목적의 데이터 제공
- 관리와 책임에 영향을 주는 변경만 이력으로 남김

---

## 테스트 결과 API

- 테스트 실행 결과를 기록
- 하나의 테스트 케이스는 여러 번 실행될 수 있음
- 테스트 결과 등록
- 테스트 결과 목록 조회

연관 테이블:

- TestResult

ETC.

- 테스트 결과는 “기록”
- 테스트 케이스의 현재 상태와 분리됨

---

## 첨부 파일 API

- 테스트 케이스 또는 테스트 결과에 대한 증빙 파일 관리
- 첨부 파일 업로드
- 첨부 파일 목록 조회

연관 테이블:

- Attachment

ETC.

- 특정 엔티티에 종속되지 않는 첨부 구조
- 추후 댓글, 이슈 등으로 확장 가능성이 있어 해당 방식으로 관리

---

## 대시보드 API

- 회의 및 관리용 요약 데이터를 제공
- 조회 전용 API
- 프로젝트 전체 테스트 현황 조회
- 테스트 케이스 상태별 현황 조회
- 담당자별 테스트 진행 상황 조회
- 마감일이 지난 테스트 케이스 목록 조회

ETC.

- CRUD와 분리된 집계 전용 API
- 회의에서 바로 사용할 수 있는 데이터 제공

---

## API 설계 요약

- 모든 권한 판단은 프로젝트 단위
- 상태, 담당자 변경은 이력으로 분리 관리
- 테스트 상태는 프로젝트별 커스터마이징 가능
- 회의 중심 사용을 고려한 대시보드 API 제공
